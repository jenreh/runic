# FalkorMigrate — Claude Code Prompts (Phase 0)

---

## Phase 0 — MVP Migration Runner

### Task

Build `falkormigrate`, a new standalone Python package that implements a lightweight,
Alembic-style migration framework for FalkorDB graph databases. Phase 0 delivers the
minimal viable runner: version tracking via a metadata node inside the graph, a linear
(single-head) revision chain, a small `op.*` operations API, and the four CLI commands
needed to create and run migrations (`init`, `revision`, `upgrade`, `downgrade`,
`current`).

This is a **greenfield package** — there is no existing codebase to integrate into.
Scaffold the full package from scratch.

---

### Context

**Target database:** FalkorDB (Redis-based graph DB, OpenCypher dialect).
**Python client:** `falkordb` (PyPI package `FalkorDB`, ≥ 1.6.1).
Key client surface:

```python
from falkordb import FalkorDB

db = FalkorDB(host="localhost", port=6379)
g = db.select_graph("my_graph")
g.query("CREATE INDEX FOR (n:Person) ON (n.email)")
g.ro_query("MATCH (v:_FalkorMigrateVersion) RETURN v.revision")
db.execute_command(
    "GRAPH.CONSTRAINT",
    "CREATE",
    "my_graph",
    "UNIQUE",
    "NODE",
    "Person",
    "PROPERTIES",
    "1",
    "email",
)
db.execute_command("GRAPH.CONSTRAINT", "DROP", ...)
```

**Tooling stack (match existing project conventions):**

- Python **3.12+** (use 3.12 minimum; do not require 3.14-only features).
- Dependency/env management: **uv** (`pyproject.toml` with `[project.dependencies]`).
- Task runner: **Taskfile.yml** with tasks `format`, `lint`, `test`, `build`.
- Formatter/linter: **Ruff** (line length 88, rules `E`, `F`, `I`, `UP`, `B`).
- Test runner: **pytest** with **pytest-cov** (coverage target ≥ 80 %).
- Template engine: **Mako** (`mako` PyPI package).
- CLI framework: **Typer** (`typer[all]`).
- Logging: parameterized `log.info("msg: %s", val)` — **no f-strings in log calls**.
- Type annotations on every function and method.
- No file may exceed **1000 lines** — extract classes/modules if needed.
- Commits: Conventional Commits (`feat:`, `fix:`, `refactor:`, `test:`, `chore:`).

**Atomicity constraint (critical):** FalkorDB has no multi-statement transaction over
the graph and no transactional DDL. A migration that runs several `op.*` calls is NOT
atomic. The version node is stamped **only after all steps in `upgrade()` / `downgrade()`
succeed**. A mid-migration failure leaves the prior revision stamped and must surface a
clear error with the partial state. Idempotency in data-transform helpers is mandatory.

**1. Config (`config.py`)**

- `Config` dataclass with fields: `script_location: str`, `falkordb_url: str`,
  `graph_name: str`, `version_strategy: str = "node"`.
- Load from `falkormigrate.ini` (configparser `[falkormigrate]` section).
- Support env overrides: `FALKORMIGRATE_URL`, `FALKORMIGRATE_GRAPH`.
- Expose `Config.from_ini(path: str | Path) -> Config`.
- `falkormigrate.ini` template generated by `init`:

  ```ini
  [falkormigrate]
  script_location = falkormigrate
  falkordb.url = falkor://localhost:6379
  falkordb.graph = my_graph
  version_strategy = node
  ```

**2. Version node (`version.py`)**

- `VersionNode` class wrapping a `falkordb` graph handle.
- `get() -> str | None` — `MATCH (v:_FalkorMigrateVersion) RETURN v.revision`.
- `set(revision: str) -> None` — `MERGE (v:_FalkorMigrateVersion {singleton: true})
  SET v.revision = $rev, v.applied_at = timestamp()` (parameterized).
- `clear() -> None` — sets `revision = None` (used for downgrade-to-base).
- Nodes with label `_FalkorMigrateVersion` must be excluded from data migrations.

**3. Revision script model (`script.py`)**

- `Revision` dataclass: `revision: str`, `down_revision: str | None`,
  `branch_labels: list[str]`, `depends_on: list[str]`, `irreversible: bool`,
  `snapshot: bool`, `message: str`, `create_date: datetime`, `path: Path`.
- `ScriptDirectory`:
  - `load(script_location: Path) -> ScriptDirectory` — scans `versions/`, imports
    each file, extracts module-level `revision`, `down_revision`, `irreversible`,
    `snapshot`; builds an in-memory dict.
  - `get_revision(rev_id: str) -> Revision` — supports full id or unique prefix.
  - `iterate_revisions(base_rev: str | None, target_rev: str) -> list[Revision]` —
    returns ordered list of revisions to apply for upgrade (base → target) or
    downgrade (target → base). Raises `AmbiguousRevision`, `RevisionNotFound`.
  - `generate_revision_id() -> str` — 12-char lowercase hex.
  - `create(message: str, head: str | None, script_location: Path) -> Path` — renders
    `script.py.mako` to `versions/<rev_id>_<slug>.py`, returns the new path.

**4. Operations API (`operations.py`)**

- `GraphOperations` class instantiated with `(graph, db, preview: bool = False)`.
- In preview mode, all methods **append** a string representation of the operation to
  `self.preview_log: list[str]` and do NOT execute against the DB.
- Methods to implement in Phase 0:

  | Method | Cypher/command emitted |
  |---|---|
  | `run_cypher(query, params=None)` | `graph.query(query, params)` |
  | `run_command(*args)` | `db.execute_command(*args)` |
  | `create_range_index(label, prop, *, rel=False)` | `CREATE INDEX FOR (n:Label) ON (n.prop)` |
  | `drop_range_index(label, prop, *, rel=False)` | `DROP INDEX ON :Label(prop)` |
  | `create_constraint(kind, entity, label, props)` | auto-create range index if `kind=="UNIQUE"`, then `GRAPH.CONSTRAINT CREATE`; poll `CALL db.constraints()` until `OPERATIONAL` or raise on `FAILED` |
  | `drop_constraint(kind, entity, label, props)` | `GRAPH.CONSTRAINT DROP` |

- `kind` for constraints: `"UNIQUE"` or `"MANDATORY"`.
- `entity` for constraints: `"NODE"` or `"RELATIONSHIP"`.
- Constraint polling: check `CALL db.constraints()` result, retry up to 30 × 0.5 s;
  raise `ConstraintFailedError` if status is `FAILED`, `ConstraintTimeoutError` on
  timeout.
- Expose a module-level `op: GraphOperations` proxy, bound by `MigrationContext`
  before calling `upgrade()`/`downgrade()`.

**5. Migration context (`context.py`)**

- `MigrationContext`:
  - `__init__(config: Config, preview: bool = False)` — builds `FalkorDB` connection
    from URL, `select_graph`, instantiates `VersionNode`, `ScriptDirectory`,
    `GraphOperations`.
  - `current() -> str | None` — delegates to `VersionNode.get()`.
  - `upgrade(target: str = "head") -> None`:
    1. Load current revision.
    2. Compute ordered revision list to target (ScriptDirectory).
    3. For each revision: bind `op` proxy, call `module.upgrade()`, on success call
       `VersionNode.set(revision)`. On any exception: log error with current revision,
       re-raise without stamping the failed revision.
  - `downgrade(target: str) -> None`:
    1. Refuse (raise `IrreversibleMigrationError`) if any revision in the downgrade
       path has `irreversible = True`, unless the caller has set `force=True`.
    2. Same step loop calling `module.downgrade()`, stamp each revision's
       `down_revision` (or clear on `base`).
  - `preview_mode: bool` — passed to `GraphOperations`; context collects and returns
    `op.preview_log`.

**6. CLI (`cli.py`)**
Typer app with the following commands:

- **`init <directory>`** — creates `falkormigrate.ini`, `<directory>/env.py`,
  `<directory>/script.py.mako` (copy from package templates), `<directory>/versions/`
  (empty dir with `.gitkeep`). Raises if directory already exists unless `--force`.

- **`revision -m <message>`** — calls `ScriptDirectory.create()`; prints the created
  file path. Options: `--head <rev>` (default: current head), `--rev-id <id>` (custom).

- **`upgrade <target>`** — default `head`; `--preview` prints statements without
  executing; `--ini <path>` (default `falkormigrate.ini`).

- **`downgrade <target>`** — required; `--force` to cross irreversible markers;
  `--preview`; `--ini <path>`.

- **`current`** — prints the current revision id (or `<none>` if unset) and the
  migration message from the script file.

```

branch_labels = ${repr(branch_labels)}
depends_on = ${repr(depends_on)}
irreversible = False
snapshot = False

    pass

def downgrade() -> None:
    pass
```

**8. Tests**

- `conftest.py`: pytest fixture `mock_graph` returning a `MagicMock` with the falkordb
  graph API; fixture `mock_db` for the `FalkorDB` client; fixture `tmp_versions`
  creating a temp `versions/` dir with two chained test revision files.
- `test_version.py`: `get` on empty graph returns `None`; `set` issues correct
  parameterized Cypher; `clear` sets revision to `None`.
- `test_script.py`: `load` finds both revisions; `iterate_revisions` returns correct
  order for upgrade and downgrade; prefix lookup works; ambiguous prefix raises.
- `test_operations.py`: in preview mode, no calls to graph/db; `create_constraint`
  calls `create_range_index` first for UNIQUE; polling raises on `FAILED` status.
- `test_context.py`: successful upgrade stamps version after each revision; mid-upgrade
  exception leaves previous version stamped; `downgrade` to `base` clears version.
- `test_config.py`: loads from ini; env override for URL and graph name.
