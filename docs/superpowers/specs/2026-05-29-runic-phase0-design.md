# runic Phase 0 — Design Spec

**Date:** 2026-05-29
**Status:** Approved

## Overview

`runic` is a standalone Python package implementing a lightweight, Alembic-style migration framework for FalkorDB graph databases. Phase 0 delivers the minimal viable runner: version tracking via a metadata node inside the graph, a linear (single-head) revision chain, an `op.*` operations API, and five CLI commands (`init`, `revision`, `upgrade`, `downgrade`, `current`).

This is a **greenfield package** scaffolded from scratch inside the `runic` repo.

---

## Key Design Decisions

### Package name: `runic`

The Python package and CLI tool are both named `runic`. CLI entry point: `runic`.

### Config: `env.py` only — no `.ini` file

Unlike the spec draft (which used `falkormigrate.ini`), `runic` follows a pure Python config pattern:

- `runic init <dir>` (default dir: `runic/`) generates `<dir>/env.py`.
- `env.py` is a Python script executed by the CLI on every command that needs a DB connection.
- `env.py` calls `context.configure(connection=db, graph=graph)` to set up the runtime context.
- No `.ini` file is created or required.

**Rationale:** More Pythonic; avoids a second config surface; secrets stay in environment variables or application config, not in a committed `.ini` file.

### CLI discovery of `env.py`

- Default: `./runic/env.py` (relative to cwd).
- Override: `--config <path>` on all commands that need a DB connection.

### `script_location` defaulting

`script_location` (where `versions/` lives) defaults to the **directory containing `env.py`**. The user can override it by passing `script_location=Path("…")` to `context.configure()`.

## Module Specs

### `config.py`

```python
@dataclass
class Config:
    script_location: Path
    version_strategy: str = "node"
```

No `from_ini()`. Config is built programmatically from `context.configure()` parameters. `script_location` defaults to `env.py`'s parent dir when not explicitly provided.

### `context.py` — module-level singleton

A module-level `MigrationContext` instance acts as a singleton, mirroring Alembic's `context` module:

```python
# runic/context.py
_ctx: MigrationContext | None = None

def configure(
    connection: FalkorDB,
    graph: Any,
    script_location: Path | None = None,
    version_strategy: str = "node",
) -> None: ...

def is_preview() -> bool: ...

def run_migrations() -> None: ...  # reserved for future env.py use; Phase 0 unused

def get() -> MigrationContext: ...  # returns configured context or raises
```

`MigrationContext` class:
- `configure(...)` — stores connection, graph, builds VersionNode + ScriptDirectory + GraphOperations.
- `current() -> str | None`
- `upgrade(target: str = "head") -> None`
- `downgrade(target: str, *, force: bool = False) -> None`
- `preview_log: list[str]` — populated when `preview=True`

### `version.py`

`VersionNode(graph)`:
- `get() -> str | None` — `MATCH (v:_FalkorMigrateVersion) RETURN v.revision`
- `set(revision: str) -> None` — `MERGE (v:_FalkorMigrateVersion {singleton: true}) SET v.revision = $rev, v.applied_at = timestamp()`
- `clear() -> None` — sets `revision = None`

### `script.py`

`Revision` dataclass: `revision`, `down_revision`, `branch_labels`, `depends_on`, `irreversible`, `snapshot`, `message`, `create_date`, `path`.

`ScriptDirectory`:
- `load(script_location: Path) -> ScriptDirectory`
- `get_revision(rev_id: str) -> Revision` — supports prefix lookup
- `iterate_revisions(base_rev: str | None, target_rev: str) -> list[Revision]`
- `generate_revision_id() -> str` — 12-char lowercase hex
- `create(message: str, head: str | None, script_location: Path) -> Path`

Exceptions: `AmbiguousRevision`, `RevisionNotFound`.

### `operations.py`

`GraphOperations(graph, db, preview: bool = False)`:

| Method | Action |
|---|---|
| `run_cypher(query, params=None)` | `graph.query(query, params)` |
| `run_command(*args)` | `db.execute_command(*args)` |
| `create_range_index(label, prop, *, rel=False)` | `CREATE INDEX FOR (n:Label) ON (n.prop)` |
| `drop_range_index(label, prop, *, rel=False)` | `DROP INDEX ON :Label(prop)` |
| `create_constraint(kind, entity, label, props)` | auto-creates range index if UNIQUE, then `GRAPH.CONSTRAINT CREATE`; polls `CALL db.constraints()` to OPERATIONAL or raises |
| `drop_constraint(kind, entity, label, props)` | `GRAPH.CONSTRAINT DROP` |

Preview mode: appends to `self.preview_log: list[str]`, does not execute.

Constraint polling: 30 × 0.5s retries; raises `ConstraintFailedError` on FAILED, `ConstraintTimeoutError` on timeout.

Module-level `op` proxy: bound by `MigrationContext` before calling `upgrade()`/`downgrade()`.

### `cli.py`

All commands accept `--config` (default: `./runic/env.py`).

**Commands split into two groups by execution model:**

**No-DB commands** (`init`, `revision`): derive `script_location = config_path.parent`; do NOT execute `env.py` (avoids requiring a live FalkorDB for offline tasks).

**DB commands** (`upgrade`, `downgrade`, `current`): execute `env.py` via `exec()` in an isolated namespace; `env.py` calls `context.configure(...)` which stores the connection and graph on the module-level singleton; CLI then dispatches the command.

| Command | DB needed | Behavior |
|---|---|---|
| `init [dir]` | No | Creates `<dir>/env.py` (from template), `<dir>/script.py.mako`, `<dir>/versions/.gitkeep`. Default dir: `runic`. Raises if exists unless `--force`. |
| `revision -m <msg>` | No | `script_location = config_path.parent`. Calls `ScriptDirectory.create()`; prints created path. Options: `--head`, `--rev-id`. |
| `upgrade [target]` | Yes | Default `head`. Executes `env.py` → `context.configure()`. `--preview` prints without executing. |
| `downgrade <target>` | Yes | Required target. Executes `env.py`. `--force` to cross `irreversible` markers. `--preview`. |
| `current` | Yes | Executes `env.py`. Prints current revision id (or `<none>`) and message. |

### Templates

**`templates/env.py.mako`** (generated by `runic init`):
```python
import os
from runic import context
from falkordb import FalkorDB

FALKORDB_URL = os.getenv("FALKORDB_URL", "falkor://localhost:6379")
FALKORDB_GRAPH = os.getenv("FALKORDB_GRAPH", "my_graph")

db = FalkorDB.from_url(FALKORDB_URL)
graph = db.select_graph(FALKORDB_GRAPH)
context.configure(connection=db, graph=graph)
```

**`templates/script.py.mako`** (migration files):
```python
"""${message}

Revision ID: ${up_revision}
Revises: ${down_revision}
Create Date: ${create_date}
"""
from runic import op

revision = ${repr(up_revision)}
down_revision = ${repr(down_revision)}
branch_labels = ${repr(branch_labels)}
depends_on = ${repr(depends_on)}
irreversible = False
snapshot = False

def upgrade() -> None:
    pass

def downgrade() -> None:
    pass
```

## Atomicity Contract

- Version node stamped **only after all steps in `upgrade()`/`downgrade()` succeed**.
- Mid-migration failure leaves prior revision stamped; logs partial state clearly.
- `irreversible = True` blocks `downgrade` unless `force=True`.

## Tests

- `conftest.py`: `mock_graph` (MagicMock falkordb graph), `mock_db` (MagicMock FalkorDB client), `tmp_versions` (temp dir with two chained revision files).
- `test_version.py`: get on empty → None; set issues correct Cypher; clear sets None.
- `test_script.py`: load finds both revisions; iterate_revisions correct order; prefix lookup; ambiguous prefix raises.
- `test_operations.py`: preview mode → no graph/db calls; UNIQUE create_constraint calls create_range_index first; polling raises on FAILED.
- `test_context.py`: upgrade stamps version after each revision; mid-upgrade exception leaves previous stamped; downgrade to base clears.
- `test_config.py`: Config dataclass initializes correctly; script_location resolves.

Coverage target: ≥ 80%.
