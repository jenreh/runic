## A. Security

- [x] **S1 🔴 AGE SQL injection via `graph_name`** — `runic/ogm/driver/age.py:518,525` (and the migrate AGE path). `self._graph_name` is interpolated into the `cypher('{graph}', …)` SQL with f-strings (`noqa: S608`). No validation. Fix: validate the graph name against `^[A-Za-z_][A-Za-z0-9_]*$` at driver construction (and/or quote via `psycopg.sql.Identifier`).
- [x] **S2 🔴 FalkorDB index-DDL Cypher injection** — `runic/migrate/adapters/falkordb.py:201-264` (`create_range_index`, `create_fulltext_index`, `create_vector_index`, `drop_fulltext_index`). `label`, `prop`, `language`, `stopwords`, `similarity` interpolated into Cypher without escaping. Fix: shared identifier validator + Cypher string-literal escaper; allow-list `language`/`similarity`.
- [x] **S3 🔴 Migration-script code injection via autogen** — `runic/migrate/autogen.py:59-105`. DB-derived identifiers (label, prop, language, stopwords) are rendered into generated `.py` migration source with bare `f'"{x}"'` instead of `repr()`. A crafted identifier breaks out → code in the generated migration. Fix: use `!r` / `repr()` for every interpolated value.
- [x] **S4 🔴 Migration-script code injection via `message`** — `runic/migrate/script.py` (`new_revision` → Mako `${message}` inside the docstring). A `message` containing `"""` breaks the docstring and injects code that runs on import. Fix: sanitize/escape `message` before rendering (strip/escape triple-quotes, or pass via a context that can't break out).
- [x] **S5 🟠 Path traversal → arbitrary file exec via `.runic`** — `runic/migrate/cli.py:37`. `.runic` content is read as a path and `exec()`-ed (`cli.py:77`) with no containment check. Fix: resolve and require the path to be within the project root; reject symlinks; require regular file.
- [x] **S6 🟠 OGM identifier injection (systematic)** — labels & relationship types interpolated unparameterized across `runic/ogm/query/builder.py:590,817-830`, `runic/ogm/mapper/relationship_writer.py:135,137,158-161`, `runic/ogm/mapper/relationship_loader.py`, `runic/ogm/mapper/mapper.py`, and the dialect `fulltext_call()` in `runic/ogm/driver/falkordb.py:91-95` & `neo4j.py:59-64`. Fix: validate `primary_label`/labels in `Node.__init_subclass__` and relationship type in `Relation()` against a safe identifier pattern (single chokepoint).
- [x] **S7 🟠 OGM `operations.py` label/property injection (public API)** — `runic/ogm/operations.py:46,70,72` (`rename_property`, `relabel_nodes`). Documented for use outside migrations → reachable with untrusted identifiers. Fix: validate label/property names against the safe pattern before interpolation.
- [x] **S8 🟡 `project()` raw-string injection** — `runic/ogm/query/builder.py:500` → `_compiler.py:133`. Raw strings flow unvalidated into the `RETURN` clause. Fix: restrict string projections to a `alias.prop` pattern (or document as a power-user escape hatch and validate).
- [x] **S9 🟡 Debug logging leaks property values** — `runic/ogm/session/session.py:71`, `runic/ogm/session/async_session.py:57`. With `log_cypher=True`, full `params` (incl. PII/secrets) are logged. Fix: document the risk in the flag's docstring (and/or redact).

## B. Bugs

- [x] **B1 🔴 FalkorDB `GRAPH.CONSTRAINT` passes `label` instead of graph key** — `runic/migrate/adapters/falkordb.py:288,321`. 3rd arg must be `self._graph.name`. Constraint create/drop currently malfunction. Fix: pass the graph key.
- [x] **B2 🟠 `_poll_constraint` doesn't filter to the target constraint** — `runic/migrate/adapters/falkordb.py:298-311`. Returns/raises on the first constraint of *any* label/props. Fix: match `label`+`props` before reading status.
- [x] **B3 🟠 Fulltext multi-property mismatch in `read_live_schema`** — `runic/migrate/introspect.py:475-482`. Emits one `FulltextIndex` per prop while `introspect_graph` groups them → false-positive diffs. Fix: group fulltext props by label.
- [x] **B4 🟠 `fork()` hardcodes encryption** — `runic/migrate/adapters/neo4j.py:74-80` (`encrypted=True`) and `memgraph.py:79` (`encrypted=False`), ignoring the original setting → silent security-posture change. Fix: derive encryption from `self._driver.uri` scheme.
- [x] **B5 🟠 `get_existing_specs` swallows exceptions** — `runic/migrate/adapters/neo4j.py:215-216,232-233` and `memgraph.py:198-199,216-217`. Bare `except Exception` → empty specs → autogen treats all indexes as missing. Fix: re-raise unexpected errors (mirror FalkorDB's selective pattern).
- [x] **B6 🟡 AGE adapter has no rollback on error** — `runic/migrate/adapters/age.py:59-72`. `execute`/`run_query` commit with no try/finally; a failure leaves the txn aborted. Fix: rollback on exception.
- [x] **B7 🟠 `count()` / async `count()` corrupt builder state on error** — `runic/ogm/query/builder.py:737-768`, `runic/ogm/query/specialised.py:75-100`. State saved/mutated/restored with no try/finally. Fix: wrap in try/finally.
- [x] **B8 🟡 `IN`/`NOT IN` ignore `cypher_fn` wrapper** — `runic/ogm/query/_compiler.py:100`. Uses `${param_name}` instead of computed `param_ref` → wrong Cypher for vector/geo fields. Fix: use `param_ref`.
- [x] **B9 🟠 Connection leak in `create_age_driver`** — `runic/ogm/driver/age.py:574-582`. If `_setup_age_connection` raises, `conn` is never closed. Fix: close on failure before re-raise.
- [x] **B10 🟠 Bolt session leak in `BoltDriver.begin()`** — `runic/ogm/driver/bolt.py:139-140`. If `begin_transaction()` fails, the opened session is never closed. Fix: close session on failure.
- [x] **B11 🟠 Duplicate CREATE on exception in `_flush_pending`** — `runic/ogm/session/session.py:511-529` and `async_session.py:398-414`. `_pending.clear()` only at loop end → reprocessed entities on retry (durable dup on FalkorDB). Fix: remove each entity as it succeeds.
- [x] **B12 ⚪/🟡 Duplicate DELETE on exception in `_flush_deleted`** — `runic/ogm/session/session.py:552-564` and `async_session.py:434-444`. Same pattern as B11. Fix: remove each entity as it succeeds.
- [x] **B13 🟡 Tracking sets cleared before `driver.commit()`** — `runic/ogm/session/session.py:155-156`. On commit failure, the session has lost what to retry. Fix: clear after a successful driver commit.
- [x] **B14 🟡 Async `__aexit__` lacks try/finally** — `runic/ogm/session/async_session.py:362-372`. If commit/rollback raises, `close()`/expunge never runs. Fix: move close into finally.
- [x] **B15 🟡 `EnumConverter.from_graph` crashes on bad value** — `runic/ogm/core/types.py:73-76`. Unhandled `ValueError`. Fix: wrap and raise a descriptive error.
- [x] **B16 🟡 `DatetimeConverter.from_graph` crashes on bad ISO string** — `runic/ogm/core/types.py:47-54`. Unhandled `ValueError`. Fix: wrap and raise a descriptive error.
- [ ] **B17 🟡 `Field(default=[])` shared mutable state** — `runic/ogm/core/descriptors.py:318-364` / `get_default`. Mutable default shared across instances. Fix: detect mutable defaults and raise, steering to `default_factory`.
- [ ] **B18 ⚪ `metadata.snapshot()` shallow copy** — `runic/ogm/core/metadata.py:155-167`. Snapshot shares mutable `NodeMeta`/`EdgeMeta`. Fix: deepcopy on snapshot/restore.
- [x] **B19 🟡 Index-priority `if` vs `elif`** — `runic/ogm/schema/index_manager.py:51`. A field with `unique=True` + `index_type` emits two specs. Fix: `elif`.
- [x] **B20 🟡 `get_revision_message` swallows all exceptions** — `runic/migrate/context.py:113-117`. Fix: catch specific revision errors / log with `exc_info`.

## Deferred (needs sign-off — user opted to skip)

These three carry API/regression/perf risk disproportionate to their value, and were
explicitly deferred:

- **C3 🟠 Repository / AsyncRepository duplication** — the verifier judged it not
  consolidatable without losing sync/async type safety; large regression surface for a DRY
  nicety.
- **B17 🟡 `Field(default=[])` raise on mutable default** — making it raise is a public-API
  behavior change; the documented guidance is already `default_factory`. Left as a
  documented footgun rather than a hard error.
- **B18 ⚪ `metadata.snapshot()` deepcopy** — adds a deepcopy cost on every snapshot for a
  bug nothing currently triggers (snapshot is always taken after `finalize()`).

## Final code review (caught 2 real issues, now fixed)

An independent review of the diff surfaced two genuine bugs the initial pass missed — both
pre-existing latent bugs in the constraint code that B1/B2 had only partially addressed:

- **drop_constraint graph key** — `create_constraint` was fixed to pass `self._graph.name`
  at arg 3, but `drop_constraint` still passed `label`. Now fixed (and asserted in tests).
- **`_poll_constraint` row layout** — the poll dereferenced `row[0]` as if rows were nested,
  but `FalkorDBResult.rows` is the flat `result_set` (each row = `[type, label, props,
  entity, status]`, matching `introspect.read_live_schema`). The old behaviour always timed
  out against real FalkorDB; the unit test masked it with an unrealistic nested mock. Now
  uses flat indexing, and the mocks were corrected to realistic flat rows.
